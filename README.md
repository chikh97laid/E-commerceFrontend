A modern, responsive, and feature-rich **E-Commerce Frontend** built with **React.js**.  
The project follows professional front-end architecture and clean code principles while integrating seamlessly with an **ASP.NET Core Web API**.

---

# 🚀 Tech Stack

- ⚛️ **React.js**
- ⚡ **Vite**
- 🎨 **Tailwind CSS**
- 🧭 **React Router (Loaders & Actions)**
- 🌐 **Axios**
- 🔐 **JWT Authentication**
- 📦 **REST API Integration**

---

# ✨ Features

## 🏗️ Clean Architecture

- Modular and reusable component structure.
- Separation of concerns for maintainability.
- Reusable layouts, forms, and UI components.
- DRY (Don't Repeat Yourself) principles.

---

## ⚡ Optimized Data Loading

- Implemented **React Router Loaders**.
- Prefetches data before rendering pages.
- Eliminates repetitive `useEffect` data fetching.
- Cleaner routing and better user experience.

---

## 🔐 Authentication & Authorization

- JWT Authentication.
- Automatic token injection using Axios interceptors.
- Secure session management.
- Protected routes.
- Role-Based Access Control (RBAC).
- Admin-only dashboard and pages.

---

## 🛡️ Session Management

- Detects expired sessions.
- Intercepts **401 Unauthorized** responses.
- Automatically logs out users.
- Redirects to Login page securely.

---

## 📊 Advanced UI Features

- Global Search
- Dynamic Filtering
- Client-side Pagination
- Responsive Design
- Optimized Rendering

---

## 📝 Form Handling

Supports:

- Standard Forms
- Image Uploads
- `multipart/form-data`
- Validation
- File Preview

# ⚙️ Getting Started

## Prerequisites

Install:

- Node.js
- npm

---

## Clone Repository

```bash
git clone https://github.com/your-username/InfoBook-Frontend.git

cd InfoBook-Frontend
```

---

## Install Dependencies

```bash
npm install
```

---

## Configure Environment Variables

Create a `.env` file in the project root.

```env
VITE_API_URL=https://your-api-domain.monsterasp.net
```

---

## Run Development Server

```bash
npm run dev
```

Open:

```
http://localhost:5173
```
